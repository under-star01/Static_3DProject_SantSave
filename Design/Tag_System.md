# Tag System (캐릭터 교체 시스템)

![이미지 설명](Design/Images/TestImage.jpg)

## 1. 목적
- 플레이어가 3명의 캐릭터를 **쿨타임 기반으로 교체(Tag)** 하며 전투 리듬을 만들도록 유도
- 캐릭터별 역할(딜/서포트/컨트롤)을 상황에 맞게 활용

---

## 2. 용어 정의
- **Active Character**: 현재 조작 중인 캐릭터
- **Standby Character**: 대기 중(교체 가능한) 캐릭터
- **Tag Cooldown**: 태그(교체) 후 다시 태그할 수 있을 때까지의 대기 시간
- **Invincibility (I-Frame)**: 태그 직후 잠깐의 무적 시간

---

## 3. 기본 규칙 (MVP)
### 3.1 태그 입력
- 키보드: `1 / 2 / 3` (각 캐릭터로 즉시 전환)

### 3.2 태그 쿨타임
- 태그 쿨타임: **8.0초**
- 쿨타임은 **태그 실행 시점**부터 카운트 시작

### 3.3 태그 무적
- 태그 후 무적 시간: **0.5초**
- 무적 중에는 피격/넉백/상태이상 적용되지 않음

---

## 4. 제한 조건
### 4.1 태그 불가 조건
- 피격 경직 상태(또는 다운 상태)
- 사망 상태
- 특정 보스 패턴(강제 연출) 진행 중

### 4.2 태그 가능 조건
- 전투/이동 상태에서 언제든 가능 (단, 불가 조건 제외)
- 대기 캐릭터가 생존 상태일 것

---

## 5. 위치 / 카메라 처리
- 전환 시 **기존 Active Character 위치에서 교체**
- 카메라는 **플레이어(Active)만 추적**
- 태그 순간 카메라 “튐” 방지:
  - 교체 시 카메라 목표가 끊기지 않도록 1프레임 내 교체 완료

---

## 6. UI 표시
- 캐릭터 슬롯(1/2/3) 표시
- 태그 쿨타임:
  - 선택된 슬롯에 **쿨타임 게이지(원형/바)** 표시
  - 남은 시간 숫자 표시(선택)

---

## 7. 예외 / 에지 케이스
- 태그 대상 캐릭터가 사망이면:
  - 태그 실패 + UI에 "X" 표시 (또는 자동 스킵)
- 태그 입력 연타 시:
  - 태그 쿨타임 중이면 무시 (입력 버퍼 사용 여부는 추후 논의)
